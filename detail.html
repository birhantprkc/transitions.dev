Transitions.dev is a curated collection of the most essential UI transitions for web apps: dropdowns, modals, notification badges, number pop-ins, page slides, accordions and more. Each one is a small, self-contained piece of motion you can drop into any project.

Every transition is built with plain CSS (plus a little JavaScript only where a demo needs it) and tuned against a shared scale of motion tokens (duration, easing, distance, blur and scale), so they feel consistent no matter where you use them.

Pick a transition from the sidebar to see it three ways: a live Preview, the portable CSS, and a self-contained React version. When you find one you like, see Installation for the ways to bring it into your project.

Copy the snippet

There's nothing to install: open a transition, copy the CSS (or React) from the code panel, and paste it into your project; snippets are namespaced (.t-*) with their motion tokens exposed as CSS variables you can tweak directly.

Install with the CLI (npm)

Pull the transitions straight into your project from the terminal, no manual copy. Free transitions work with no account.

Free version
npx transitions-pro add --all

Writes every free transition into ./transitions/. Run npx transitions-pro list to see everything available.

Pro version
npx transitions-pro add --pro

Pro: grabs everything including the Pro CSS and React. The first time, it opens your browser to confirm a short code (no password, no API key in your terminal), then pulls it all in. Sign out any time with logout. See Transitions Pro.

Install with the Skill & Refine tool

Prefer not to copy by hand? Two tools deliver transitions straight into your codebase.

Free version
npx skills add Jakubantalik/transitions.dev

Skill: installs the full transition library. Works with Claude Code, Cursor, GitHub Copilot, Codex and Gemini CLI. Ask your agent for a transition by name and it drops the right snippet into your files.

Pro version
npx transitions-pro skill

Pro Skill: installs the premium transitions as an agent skill (CSS, React, TypeScript). Opens your browser once to confirm your Pro account, then drops it into your skills directory. See Transitions Pro.

Free & Pro version
npx transitions-refine live

Refine tool: run in your terminal to open a live panel on your running app. Scans your existing motion, aligns it to the token scale, and lets you fine-tune duration, easing and stagger visually, then apply the result back to your code.

Motion should never get in the way. Every transition here stays accessible by default.

  • Respects reduced motionEvery transition honors @media (prefers-reduced-motion: reduce). When a visitor has asked their system to minimize motion, animations and transitions are switched off or collapsed to an instant state change, with no movement, blur or scale, while the functional result (menu open, value updated, panel shown) still happens.
  • Nothing depends on motion aloneAnimation is decorative. State is always reflected in the DOM and text, never conveyed only through movement, so screen-reader and reduced-motion users get the same information.
  • Keyboard and focusInteractive demos use real <button> elements with correct aria states, keep native focus outlines, and are fully operable from the keyboard.
  • Smooth by designTransitions stick to compositor-friendly properties, namely transform, opacity and filter, and use will-change sparingly, so motion stays smooth without taxing lower-end devices.
